The Five Whys

  • Why? Because every AI agent framework today ships a 200-package dependency tree and a 200 MB Python venv. odek is an ~8 MB static binary with 2 indirect dependencies — go-vector and golang.org/x/net.
  • Why? Because you shouldn't need a PhD in Python packaging just to run go test. odek installs in 5 seconds with a single go install command — no Docker, no venv, no npm, no sagas.
  • Why? Because locking into one LLM provider is a strategic mistake. odek is provider-agnostic — Deepseek, OpenAI, Anthropic, Ollama, vLLM, Groq, Together, Fireworks. Any OpenAI-compatible endpoint.
  • Why? Because agents need real tools, not abstractions on top of abstractions. odek gives you read_file, write_file, patch, search_files, shell, and browser — all gated by a unified security layer. No plugins. No SDKs. No class hierarchies.
  • Why? Because the agent ecosystem has split into two worlds — tools that serve MCP (Claude Code, Cursor) and tools that consume MCP. odek does both in one binary. Serve your tools to any MCP client, or connect to external MCP servers for Playwright, Fetch, databases, and more.

What you get

🔒

Sandboxed Execution

odek run --sandbox — every session spawns an isolated Docker container. No network, no host mounts beyond the working directory, zero capabilities, destroyed on exit.

🧩

Sub-Agent Delegation

Parallel OS-process sub-agents via delegate_tasks. True isolation — each sub-agent is a fresh process with its own config, tools, and timeout. Up to 8 concurrent workers.

🧠

Self-Learning Skills

Detects multi-step procedures, error recoveries, and corrections during your sessions. Saves them as reusable SKILL.md files. On by default — --no-learn to disable.

💾

Persistent Memory

Three tiers: facts, session buffer, and episodes. Merge-on-write via go-vector cosine similarity. Saves ~80% LLM calls on repeated topics.

🔗

MCP Two-Way

Server: odek mcp exposes native tools to Claude Code, Cursor, any MCP client. Client: odek connects to external MCP servers and makes their tools available to the agent.

🌐

Web UI

odek serve — browser-based agent with @ resource completion, WebSocket streaming, token economics, inline loading, and a full IDE-style console. Built from Go's embed — zero npm.

🏗️

Layerable Config

Four-layer priority chain: global → project → KODE_* env vars → CLI flags. ${VAR} substitution in config files. Sensible defaults at every layer.

🔍

Native Tools

Built-in read_file, write_file, search_files, patch, shell, and browser. All gated by a unified security layer — classify operations as allow / deny / prompt per risk class.

Install in 5 seconds

  • 1 Have Go 1.24+ installed. If not: brew install go or download.
  • 2 Install:
    go install github.com/BackendStack21/kode/cmd/odek@latest
  • 3 Set your API key: export DEEPSEEK_API_KEY=sk-... or OPENAI_API_KEY
  • 4 Run your first task: odek run "How many lines in go.mod?"

One-line binary install

No Go toolchain? Auto-detects your OS and architecture: 

# Download to /usr/local/bin (may need sudo)
curl -sfLo /usr/local/bin/odek https://github.com/BackendStack21/kode/releases/latest/download/odek-$(uname -s | tr '[:upper:]' '[:lower:]')-$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/') && chmod +x /usr/local/bin/odek

# Or to current directory
curl -sfLO https://github.com/BackendStack21/kode/releases/latest/download/odek-$(uname -s | tr '[:upper:]' '[:lower:]')-$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/') && chmod +x odek-* && ./odek-* version

Cheatsheet

Commands

  • odek run <task>Single-shot task
  • odek run --session <task>Save conversation as session
  • odek continue [--id <id>] <task>Resume a saved session
  • odek replInteractive multi-turn REPL
  • odek session listList recent sessions
  • odek session show [id]View session transcript
  • odek session delete <id>Delete a session
  • odek session cleanup <days>Delete old sessions
  • odek skill listList available skills
  • odek skill import <uri>Import skill from URL or file
  • odek skill curateAudit skill quality and overlap
  • odek serve [--addr :8080]Start Web UI server
  • odek subagent --goal <s>Run a focused sub-task
  • odek mcp [--sandbox]Start MCP server
  • odek init [--global]Create config file

Key Flags

  • --model <name>LLM model (deepseek-v4-flash, gpt-4o, etc.)
  • --base-url <url>API endpoint URL
  • --sandboxRun in Docker sandbox
  • --thinking <level>Reasoning depth (enabled/disabled/low/medium/high)
  • --learnEnable skill learning (on by default)
  • --no-learnDisable skill learning
  • --max-iter <n>Max think→act cycles (default 90)
  • --system <prompt>Override system prompt
  • --no-colorDisable colored output
